Assignment 2 and Assignment 3 complete the same overall task which is reading product data from a CSV file, transforming it according to the assignment rules, and writing the results to an output file. The major difference between the two implementations is in the design structure. In Assignment 2, all functionality was placed inside a single class. That class handled reading the file, parsing records, applying transformation logic, tracking counts, and writing the output. While it worked correctly, the design was tightly coupled and harder to read because multiple responsibilities were combined in one place.

In Assignment 3, the structure was reorganized into multiple public classes, each dedicated to a specific role. For example, ProductRecord represents the data model for a product, ProductCSVReader handles reading and parsing the input file, ProductCSVWriter manages writing the output, and ETLPipeline controls the overall workflow. This separation makes the design more modular and easier to follow. Instead of one large file containing everything, each class has a clear and focused purpose, which improves readability and organization.

Assignment 3 is more object-oriented because it emphasizes the use of classes and objects to model the program’s components. The ProductRecord class is an example of encapsulation, as it bundles product data together and controls how that data is accessed. Each class represents a specific concept or responsibility, demonstrating the use of objects and abstraction. Although inheritance and polymorphism were not required for this assignment, the structure still reflects core object-oriented principles such as separation of concerns and clear class responsibilities.

The output file generated by Assignment 3 matched the output from Assignment 2, confirming that the refactored design preserved the original functionality. By running the program and verifying that the results were identical, I confirmed that Assignment 3 maintains the same behavior while improving the overall structure and design.
